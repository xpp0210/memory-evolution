# Memory Evolution v5.0 规划

> 基于安宝系统实战经验，下一代记忆进化体系
> 版本：5.0.0 | 规划日期：2026-04-04

---

## 核心升级方向

### 1. MemOS 深度集成

当前 v4.0 是独立脚本系统，与 MemOS 无协同。

**v5.0 目标：**
- MemOS 作为底层数据源（memory_search + chunk storage）
- memory-evolution 作为上层进化引擎（reflect → capture → evolve）
- 双向同步：MemOS chunk ↔ memory-evolution patterns
- 共享 skill 库：`memos-local/skills-store/` ↔ `memory/skill-bank/`

**关键集成点：**
| v4.0 | v5.0 |
|--------|--------|
| 独立检索（triple-fusion.py）| 调用 MemOS `memory_search` |
| 本地 JSON 存储 | MemOS SQLite + 向量库 |
| 手动 skill 管理 | Memory Viewer + 自动进化 |
| 批处理（cron）| 实时触发（hook）|

---

### 2. LLM 驱动的智能决策

当前 v4.0 是规则引擎（if/else 逻辑），不够灵活。

**v5.0 目标：**
- **reflect.sh 升级为 reflect-llm.py** - 用 LLM 判断任务类型/归因
- **skill-evolve.py 升级为 skill-llm-evolve.py** - 用 LLM 生成变异
- **night-consolidate.py 升级为 memory-llm-distill.py** - 用 LLM 做知识蒸馏

**新增能力：**
```
任务类型判断（LLM） → 选择进化策略
- 新任务 → CAPTURED 模式
- 失败任务 → FIX + 归因分析
- 重复任务 → DERIVED 模式
- 成功任务 → 经验模式提取

智能归因（LLM）→ 失败时自动分析 root cause
- 从代码/错误/日志中提取关键线索
- 对比 capability-map.json 的历史失败
- 生成具体修复建议
```

---

### 3. Hook 驱动的实时监控

当前 v4.0 是批处理（cron job），缺少实时性。

**v5.0 目标：**
- **新增 hooks/evolve-monitor/** - 5 个 Hook 集成
- 任务完成时立即触发进化（不等夜间整合）
- 失败时立即归因（不等 reflect.sh 手动调用）
- 成功时立即捕获模式（不等 skill-capture-scan.py 扫描）

**Hook 列表：**

| Hook | 触发条件 | 动作 |
|------|----------|------|
| `session:compact:after` | 会话压缩率 > 40% | 立即蒸馏知识 |
| `task:completed` | 任务完成 | 立即评估是否生成 skill |
| `task:failed` | 任务失败 | 立即 LLM 归因 |
| `message:preprocessed` | 消息包含学习信号 | 写入 learn-queue/ |
| `agent:bootstrap` | Agent 引导 | 热身用户画像 |

---

### 4. 多 Agent 协作进化

当前 v4.0 是单 Agent 系统，不支持团队协作。

**v5.0 目标：**
- **Team Evolution** - 多 Agent 共享能力图谱和技能库
- **Conflict Resolution** - 冲突的 skill 自动合并
- **Consensus Learning** - 多 Agent 投票选择最优进化策略

**新增模块：**
```
scripts/team-evolve.py          # 团队进化协调器
memory/team-capability-map.json  # 团队能力汇总
memory/skill-merges.json         # 技能合并历史
```

---

### 5. 自适应学习策略

当前 v4.0 是固定策略（Ebbinghaus 衰减、规则淘汰）。

**v5.0 目标：**
- **meta-learn.py 升级为 adaptive-strategy.py** - 自动测试不同策略
- **A/B Testing** - 实时比较策略效果
- **动态调整** - 根据任务成功率自动调整参数

**可优化参数：**
| 参数 | v4.0 | v5.0 |
|------|--------|--------|
| 衰减半衰期 | 固定 14 天 | 自适应（7-28天） |
| 归因阈值 | 固定 ≥3 次失败 | 自适应（2-5次） |
| 技能质量阈值 | 固定 ≥6 分 | 自适应（5-8分） |
